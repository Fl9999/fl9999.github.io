参数 说明 示例

使用 node_exporter 的 collector.textfile.directory 采集自定义数据

node_exporter 不支持直接采集主机上的端口监听状态数据。但 node_exporter 支持从本地目录直接读取指标数据。

为了实现此需求，可以写个定时脚本周期性的采集端口状态数据到本地文件中，然后配置 node_exporter 从本地文件中读取统计信息供给 Prometheus 抓取。

参考以下脚本监控指定端口的状态，并将状态数据写入指定的文件，此处为 /opt/prometheus/data/open_ports.prom

  • # TYPE node_exporter_open_ports gauge 指定 Prometheus 数据类型为 gauge。同一个指标类型只能定义一次，如果有多个 # TYPE node_exporter_open_ports gauge 定义，node_exporter 不会抓取指标而是输出错误：msg="failed to collect textfile data" file=open_ports.prom err="failed to parse textfile data from \"/opt/prometheus/data/open_ports.prom\": text format parsing error in line 3: second TYPE line for metric name \"node_exporter_open_ports\", or TYPE reported after samples"
  • 指标记录中的字符串格式的值要用 ""，不能使用 '' 。否则会报错： caller=textfile.go:227 level=error collector=textfile msg="failed to collect textfile data" file=probe_data.prom err="failed to parse textfile data from \"/opt/prometheus/data/probe_data.prom\": text format parsing error in line 5: expected '\"' at start of label value, found '\\''"

配置 node_exporter 抓取本地目录 /opt/prometheus/data/ 中的统计数据，主要是使用 textfile 这个 Collector。

在以上示例中，存在一个可能的问题： 如因某些原因，任务在某个时间点执行完成后，在之后的所有应该执行的时间点并未执行，其输出文件中的内容会保持最后一次正确执行后的内容 。在此场景下，Prometheus Server 对后续的此自定义指标抓取正常，因为每次抓取都能读取到目标文件中的数据，但实际上 抓取到的数据已经是无效数据 。

要解决此问题，可以通过监控 textfile 提供的指标 node_textfile_mtime_seconds，它记录了自定义指标输出文件的最后修改时间（时间戳）。可以监控指标输出文件的更新时间，如 10 分钟内其未更新，立即发送告警消息。
